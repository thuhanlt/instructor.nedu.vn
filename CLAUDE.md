## 1. Mục đích

`nedu-instructor` là portal dành cho **Người Dẫn Đường** (giảng viên) trong hệ sinh thái N·Education. Triết lý cốt lõi: *để người dạy chỉ lo dạy* — vận hành đẩy lịch + tài liệu + Zoom vào portal, bộ phận hỗ trợ lọc câu hỏi học viên trước, hệ thống tổng hợp phản hồi trực tiếp về tay người dạy. Portal có 9 module, 1 vai trò chính, 39 user stories.

**Người dùng = 1 vai trò duy nhất: Người Dẫn Đường.** Vận hành / bộ phận hỗ trợ / học viên là external actors — họ không dùng portal này.

## 2. Tech Stack (LOCKED — không đổi)

| Layer | Lựa chọn | Note |
|---|---|---|
| Build | **Vite 8** + React 19 + TypeScript strict | `npm create vite@latest nedu-instructor -- --template react-ts` |
| Routing | **React Router v7** (`react-router-dom`, BrowserRouter) | SPA — không SSR |
| Server state | **TanStack Query v5** (`@tanstack/react-query`) | Query keys hierarchical |
| Client state | **Zustand v5** | Cho `useAuthStore`, `usePrefsStore` (timezone/lang/sidebar) |
| Styling | **Tailwind v4** (`@tailwindcss/vite`) + CSS variables từ prototype | KHÔNG dùng Material UI / Ant Design |
| Mock API | **MSW v2** (`msw/browser`) | Toggle bằng `VITE_ENABLE_MOCKING` |
| Auth | **NLH auth-central** (Google OAuth) | Token storage keys `nlh_access_token`, `nlh_refresh_token` |
| Analytics | **GA4 + Microsoft Clarity** (`@microsoft/clarity`) | Hostname gating: chỉ fire khi `window.location.hostname === 'instructor.nedu.vn'` |
| Date | **date-fns v3** + `date-fns-tz` v3 | Cho 16 múi giờ + week/month/year view |
| Font | Playfair Display (headlines) + Inter (body) | Đã có link Google Fonts trong prototype |
| Deploy | Vercel (preview, mock=true) + Cloudflare Workers (production, mock=false) | Dual deploy bắt buộc |

**Không được dùng:** Redux, Recoil, Jotai, SWR, Axios, Prisma client, styled-components, Emotion, Material UI, Ant Design, Chakra UI. Nếu cần dropdown/select/modal phức tạp → tự build theo prototype, không add lib.

## 4. Data Model (Mock Layer + Real API contract)

Vì instructor portal là FE thuần — **không tự quản database**. DB do `api.nedu.vn` (backend Express + Supabase) quản. Portal chỉ **đọc/ghi qua API**.

Phần này mô tả **shape data** sao cho mock và real API trả về cùng format.

### 4.1. Entities Own (instructor portal CRUD trực tiếp)

| Entity | Mô tả | Owner |
|---|---|---|
| `instructor_profile` | Hồ sơ giảng viên (name, email, phone, bio, avatar, tags, timezone, language) | This portal |
| `qa_responses` | Phản hồi cho câu hỏi học viên (3 trạng thái: pending/pinned/answered) | This portal |
| `incident_reports` | Báo sự cố / thay đổi lịch (reason, action, proposed slots, note) | This portal |
| `material_requests` | Yêu cầu chỉnh sửa tài liệu (note + uploaded files) | This portal |
| `user_prefs` | localStorage: timezone, lang, sidebarOpen, qaGuideHidden, downloadedMaterials | Client-side only |

### 4.2. Entities Borrow (READ ONLY từ backend)

| Entity | Mô tả | Source |
|---|---|---|
| `programs` | 4 chương trình (Cuộc Sống / Thương Hiệu / Là Chính Mình / Sức Mạnh Vô Hạn) | ops backend push |
| `klasses` | Khoá (K1, K2, K3 cho mỗi chương trình) | ops backend push |
| `sessions` | Buổi học cụ thể (date, time, online/offline) | ops backend push |
| `materials` | File tài liệu cho từng session (kịch bản, slide) | ops backend push |
| `zoom_rooms` | Mã phòng + mật khẩu Zoom (read-only cho instructor) | ops backend push |
| `questions` | Câu hỏi học viên đã được support filter | learn.ni.sg → support → backend |
| `feedback` | Phản hồi học viên (sao + comment) | learn.ni.sg → backend |
| `class_analytics` | 4 hệ analytics phân bổ nhóm (KHÔNG có hồ sơ cá nhân) | Data Guard / metaphysical service |
| `notifications` | Thông báo (QA mới, lịch mới, ...) | backend push |

### 4.4. Response Envelope (BẮT BUỘC khớp giữa mock và real BE)

NestJS backend wrap toàn bộ response trong envelope:

```ts
// Single resource / KPI
{ data: T }

// List paginated
{ data: T[], meta: { page: number, limit: number, total: number } }

// Error (HTTP 4xx/5xx)
{ statusCode: number, message: string | string[], error: string }
```

API client (`src/shared/config/api-client.ts`) tự động unwrap `.data` cho method `get/post/patch/put`. Endpoint paginated dùng `api.getRaw()` để giữ nguyên cả `meta`.

## 6. API Contracts

Base URL: `${VITE_API_URL}/api` (prefix `/api` bắt buộc).

Auth header: `Authorization: Bearer <access_token>`.

### 6.1. Auth (qua auth-central, không phải api.nedu.vn)
- `GET  ${VITE_AUTH_CENTRAL_URL}/auth/oauth/google?return_to=<encoded>` — redirect Google OAuth
- `POST ${VITE_AUTH_CENTRAL_URL}/auth/refresh` body `{ refresh_token }` → `{ access_token, refresh_token }`
- `POST ${VITE_AUTH_CENTRAL_URL}/auth/logout` — revoke

### 6.2. Instructor profile
- `GET  /auth/me` → `{ data: AuthUser }` — gọi sau khi có token, verify JWT ở BE
- `GET  /instructor/profile` → `{ data: InstructorProfile }`
- `PATCH /instructor/profile` body `Partial<InstructorProfile>` → `{ data: InstructorProfile }`
- `POST /instructor/profile/avatar` multipart `file` → `{ data: { avatarUrl: string } }`

### 6.3. Dashboard
- `GET /instructor/dashboard?tz=<IANA>` → `{ data: DashboardData }`

### 6.4. Programs / Klasses / Sessions
- `GET /instructor/programs` → `{ data: Program[] }` (chỉ programs instructor đang dạy)
- `GET /instructor/programs/:programId/klasses/:klassId` → `{ data: { program: Program, klass: Klass, stats: { studentCount, sessionsDone, sessionsTotal, avgFeedback } } }`
- `GET /instructor/sessions?from=<YYYY-MM-DD>&to=<YYYY-MM-DD>&tz=<IANA>` → `{ data: (Session & { programName, klassLabel })[] }`
- `GET /instructor/sessions/:sessionId/feedback` → `{ data: { avgScore: number; count: number; items: FeedbackItem[] } }`

### 6.5. Materials
- `POST /instructor/sessions/:sessionId/material-requests` multipart:
  - `note: string`
  - `files: File[]` (0..N)
  → `{ data: { id: string, status: 'received' } }`

### 6.6. Q&A
- `GET /instructor/questions?program=<id|all>&klass=<id|all>&state=<state|all>` → `{ data: (Question & { programName, klassLabel, sessionTitle, deadlineHint })[] }`
- `POST /instructor/questions/:id/reply` body `{ reply: string }` → `{ data: Question }` (state = 'answered')
- `POST /instructor/questions/:id/pin` body `{ pinForSessionId?: string }` → `{ data: Question }` (state = 'pinned')
- `POST /instructor/questions/:id/pass-to-ops` → `{ data: Question }` (state = 'passed_to_ops')
- `POST /instructor/questions/:id/suggest-reply` body `{ context?: string }` → `{ data: { reply: string } }` ⭐ **AI endpoint**

### 6.7. Students analytics
- `GET /instructor/klasses` → `{ data: Array<{ programId, programName, klassId, klassLabel, type, studentCount, color }> }`
- `GET /instructor/klasses/:klassId/analytics` → `{ data: ClassAnalytics }`

### 6.8. Feedback (aggregated)
- `GET /instructor/feedback?program=<id|all>&klass=<id|all>` → `{ data: { stats: { sessions, students, avgScore, scoreSub }, items: FeedbackItem[] } }`

### 6.9. Incidents (schedule changes)
- `POST /instructor/incidents` body `IncidentReport` → `{ data: { id, status: 'received' } }`

### 6.10. Notifications
- `GET /instructor/notifications?limit=10` → `{ data: NotificationItem[] }`
- `POST /instructor/notifications/:id/read` → `{ data: NotificationItem }`

### 6.11. Error format (envelope)
```json
{ "statusCode": 401, "message": "Unauthorized", "error": "Unauthorized" }
```

API client throws `ApiError(status, message, code, details)` — UI catch và hiển thị toast.

**API Fallback rule**: nếu một endpoint fail (network / 5xx) → query hook trả `data: undefined, isError: true` — component render `<EmptyState />` với message tương ứng và nút "Thử lại" gọi `refetch()`. **Không bao giờ để trang trắng.**

## Github flow — Frontend Labs (BẮT BUỘC)

> Nguồn: `instruction-it-labs.html` (NhiLe Holdings · Team IT · Frontend Labs v1.0).
> **2 repos · 3 branches · 4 vai trò.** Áp dụng cho mọi dự án FE LABS, gồm repo này.

### Topology

- **`origin`** — repo cá nhân của nontech (vibe-code với Claude, MSW = ON).
- **git LOCAL** — máy IT: có **cả 2 remote** (`origin` + `lab`), MSW = OFF, API thật.
- **`lab`** — repo trên org NhiLe, deploy thật. 3 branch:
  - `incoming` — IT push tới đây, **KHÔNG deploy**.
  - `dev` — staging, Cloudflare auto-deploy.
  - `main` — production, protected, **chỉ IT Lead** merge.

### Vai trò (ai làm gì)

| Vai trò | PUSH | MERGE |
|---|---|---|
| Nontech Owner | `origin/main` | — |
| IT Member | `origin/main`, `lab main:incoming` | `incoming → dev` (review chéo, ≥1 IT khác) |
| IT Lead | tất cả | `dev → main` (production gate) |
| Cloudflare | — | deploy: `lab/main`→prod, `lab/dev`→staging |

### Daily flow — KHÔNG dùng feature branch ở FE

IT làm trực tiếp trên **`main` local**. Mỗi phiên:

```bash
git pull origin main            # sáng: sync code mới từ nontech
git add .
git commit -m "feat/fix: ..."
git push origin main            # sync với nontech
git push lab main:incoming      # đẩy lên lab cho IT review
```

- `main:incoming` = đẩy `main` local lên branch `incoming` của `lab` (không cần tạo branch `incoming` ở local).
- **Promotion:** `incoming → dev` cần **≥1 review từ IT khác** (KHÔNG self-review) → `dev → main` chỉ **IT Lead**.
- Staging = `lab/dev`; production = `lab/main`.

### Quy tắc

- **MSW OFF** ở môi trường IT (`VITE_ENABLE_MOCKING=false`) — plug API thật.
- `.env.local` luôn trong `.gitignore` — **KHÔNG commit**. Mỗi IT giữ env riêng; production secrets chỉ IT Lead chỉnh trên Cloudflare.
- Sau `clone`/`pull`: audit `.env` / `.gitignore`; thấy secret bị commit → báo + invalidate ngay.
- Cross-team: trước khi sửa file, báo nontech trên Slack tránh trùng; check chéo `git log` ≥ 1 lần/ngày.

> **Lưu ý nguồn API thật:** shape endpoint thật của BE là `openapi/instructor.yaml` ở repo `nedu-backend`. Mục 6 (API Contracts) bên trên là spec mock gốc — khi nối API thật, một số path/shape đã đổi (BE trả snake_case, transform trong queryFn từng hook).
